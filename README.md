# MediGuard AI — Healthcare Cybersecurity Threat Detection

> An AI-powered threat detection system for healthcare networks using anomaly detection, LSTM-based sequence modeling, and ensemble classifiers to identify cyber attacks in real time.

---

## Problem Statement

Healthcare organizations are prime targets for ransomware, unauthorized EHR access, and lateral movement attacks. Traditional signature-based tools miss zero-day threats. MediGuard AI uses machine learning to detect anomalous behavior across network traffic, EHR access logs, system events, and IoMT (Internet of Medical Things) device data.

---

## Features

- **Multi-source data ingestion** — network logs, EHR audit trails, syslog, device telemetry
- **Feature engineering pipeline** — sliding window statistics, entropy scoring, label encoding
- **Ensemble ML detection** — Isolation Forest (unsupervised) + Random Forest + XGBoost (supervised)
- **LSTM sequence model** — detects temporal attack patterns (e.g., slow brute force, data exfiltration)
- **Real-time alert engine** — severity scoring with LOW / MEDIUM / HIGH / CRITICAL levels
- **Flask dashboard** — live monitoring UI with threat timeline
- **Auto-response hooks** — block IP, quarantine device, revoke EHR session

## Threat Models Covered

| Threat | Detection Method | Dataset Feature |
|--------|-----------------|-----------------|
| Ransomware file encryption | File write rate spike | `file_write_rate`, `entropy` |
| Brute force login | Failed auth sequence | `failed_logins`, `time_delta` |
| Lateral movement | Unusual internal connections | `unique_dst_ips`, `port_scan_score` |
| Unauthorized EHR access | Off-hours access pattern | `access_hour`, `records_accessed` |
| Data exfiltration | Outbound data volume | `bytes_out`, `dst_is_external` |
| IoMT device compromise | Abnormal device polling | `device_ping_rate`, `cmd_anomaly` |

## Quickstart

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/mediguard-ai.git
cd mediguard-ai
pip install -r requirements.txt
```

### 2. Generate synthetic data

```bash
python scripts/simulate_attack.py --scenario ransomware --n_samples 5000
```

### 3. Train models

```bash
python scripts/train.py --model all --output models/saved/
```

### 4. Run the dashboard

```bash
python app.py
# Open http://localhost:5000
```

## Dataset

This project uses a **synthetic dataset** generated by `src/data/simulator.py`, modeled after:
- [CICIDS2017](https://www.unb.ca/cic/datasets/ids-2017.html) — intrusion detection benchmark
- [MITRE ATT&CK for Healthcare](https://attack.mitre.org/) — threat taxonomy

You can also plug in real network capture (PCAP) data via `src/data/preprocessor.py`.
